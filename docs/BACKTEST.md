# Backtest по всем наборам данных

Прогон `gmc-forecaster backtest` по всем сериям в `data/`. Дата: 2026-07-13.
Единица — 9 ячеек (3 продукта × 3 канала) на переход Q_t → Q_{t+1}.

Метрики: **доля-MAE** (стадия 1, процентные пункты), **объём-MAPE** (стадия 2),
**спрос-MAPE** (сквозной = доля × объём). Режимы: **realistic** (конкуренты
заморожены на Q_t — как в `forecast`) и **oracle** (фактический рынок Q_{t+1}).
Бейзлайны спроса: **персистенция** (`спрос_{t+1}=спрос_t`) и **сезонный наив**
(`спрос_t × сезонность`).

## Результаты

Модель доли обучается in-sample на `--reports` серии (усадка наклона λ выбирается
LOGO-CV; на всех подвыборках λ→пул). Столбцы спрос/бейзлайн — MAPE, %.

| серия | пер. | доля-MAE real/oracle/все-8 | объём | спрос real | спрос oracle | персист. | сез. наив |
|---|---|---|---|---|---|---|---|
| **2026-все** (11,13,31,49) | 7 | 1.20 / 1.22 / 0.91 | 5.43% | 15.90% | 15.20% | 10.32% | **8.77%** |
| гр.11 | 3 | 1.19 / 1.28 / 1.09 | 5.58% | **10.94%** | 12.28% | 14.15% | 10.49% |
| гр.13 | 2 | 1.31 / 1.26 / 0.89 | 6.35% | 19.16% | 16.69% | **8.55%** | 8.73% |
| гр.31 | 1 | 1.00 / 1.10 / 0.48 | 3.16% | 12.13% | 11.76% | 8.58% | **8.01%** |
| гр.49 | 1 | 1.21 / 1.20 / 0.88 | N/A | N/A | N/A | 4.15% | 4.46% |
| гр.2 (2019) | 3 | 0.82 / 0.85 / 0.66 | 5.72% | 13.70% | 14.55% | **11.04%** | 11.04% |

*(гр.49: объём/спрос N/A — в Q2 нет долей, объём `sold/share` не калибруется;
доля предсказывается по ценам Q2. гр.2: сезонности нет → сезонный наив =
персистенция.)*

## Ключевые наблюдения

1. **Стадия 1 (доля) сильна везде:** MAE 0.5–1.3 п.п., all-8 oracle 0.48–1.09.
   Логит воспроизводит наблюдаемые доли на всех 6 группах и обеих эрах (2017–2026),
   включая версии RU/PL/ES. Лучшая точность — гр.2 (0.66 п.п.) и гр.31 (0.48).
2. **Стадия 2 (объём) хороша:** MAPE 3–6%.
3. **Сквозной спрос — модель в целом проигрывает наиву.** Единственное исключение —
   **гр.11**: realistic 10.94% бьёт персистенцию (14.15%) и почти догоняет сезонный
   наив (10.49%). Гр.13 сильно проигрывает (19% vs 8.5%) — та же группа, где завалился
   гибрид «наив×рычаг». Это подтверждает сквозной вывод: **ценность модели в Δ
   (эффект решения), а не в абсолютном уровне** для неизменных решений.
4. **Ошибка концентрируется в крупных ячейках:** доля/объём точны по-отдельности
   (1 п.п. / 5%), но их произведение даёт 15–19% — большие ячейки доминируют MAPE.
5. **Гетерогенность по группам:** спрос-MAPE от 11% (гр.31) до 19% (гр.13);
   бейзлайны тоже разные (сез. наив 8–11%). Малые группы (31, 49) — 1 переход,
   высокая дисперсия оценки.

## Оговорки

- **In-sample:** модель доли обучается на тех же отчётах, что оценивает. Строгий
  OOS хуже (проверено ранее: train=гр.13 → test=гр.11 даёт realistic 27% vs 14%).
- **Малые выборки:** гр.31 и гр.49 — по 1 переходу (9 ячеек); гр.13 — 2. Цифры
  ориентировочные.
- **Усадка наклона:** λ=1000 (2026) / 1e6 (гр.2, одна группа) → полный пулинг;
  пер-групповые наклоны OOS не обобщаются (FINDINGS §15). global slope: −3.19
  (2026), −1.93 (гр.2).
- **Realistic vs oracle:** близки → незнание ходов конкурентов вносит малую ошибку;
  основная ошибка — структурная (share×volume), не заморозка конкурентов.

## Воспроизведение

```bash
# 2026 все группы
uv run gmc-forecaster backtest \
  --reports data/W115*.xls data/W131*.xls data/W318*.xls data/W496*.xls \
  --history data/Hst*.xlsx
# то же с разбивкой по 9 ячейкам (+ sibling <out>.percell.csv при --out)
uv run gmc-forecaster backtest \
  --reports data/W115*.xls data/W131*.xls data/W318*.xls data/W496*.xls \
  --history data/Hst*.xlsx --per-cell
# по-группно (пример гр.11)
uv run gmc-forecaster backtest --reports data/W115*.xls --history data/Hst*.xlsx
# гр.2 (2019) — без истории
uv run gmc-forecaster backtest --reports data/6.xls data/7.xls data/8.xls data/9.xls
```
