# Backtest по всем наборам данных

Прогон `gmc-forecaster backtest` по всем сериям в `data/`. Дата: 2026-07-13.
Единица — 9 ячеек (3 продукта × 3 канала) на переход Q_t → Q_{t+1}.

Метрики: **доля-MAE** (стадия 1, процентные пункты), **объём-MAPE** (стадия 2),
**спрос-MAPE** (сквозной). Режимы: **realistic** (конкуренты заморожены на Q_t —
как в `forecast`) и **oracle** (фактический рынок Q_{t+1}). Бейзлайны спроса:
**персистенция** (`спрос_{t+1}=спрос_t`) и **сезонный наив** (`спрос_t × сезонность`).

**Сквозной спрос ЗАЯКОРЕН** (с 2026-07): `спрос_t × сезонность × рычаг`, где
рычаг = модельное отношение доли (сцен/база) из логита, демпфированное
`--lever-k` (дефолт **0.5**; 1 = полный рычаг, 0 = чистый сезонный наив). Прежний
абсолютный `доля × объём` отбрасывал наблюдаемый спрос и давал 15-20% MAPE
(разбор ниже).

## Почему было 15-20%, а теперь 8% (разбор)

Ошибка спроса рождалась не там, где её мерили. Спрос = доля × объём:

- доля-MAE 1.2 п.п. выглядит «отлично», но **медианная доля своей компании
  8.2%** → 1.2 п.п. = **15.6% относительной** ошибки. Спросу важна относительная,
  а не абсолютная точность → `спрос-MAPE ≈ 15.6% (доля) + 5.4% (объём) ≈ 15.9%`.
  Стадия 2 почти ни при чём.
- Прежний прогноз **реконструировал** долю с нуля из цены/рекламы/рейтинга,
  выбрасывая наблюдаемую долю_t. Но `corr(доля_t, доля_{t+1})=0.80`,
  `corr(спрос_t, спрос_{t+1})=0.955` — величины крайне персистентны. Цена
  объясняет лишь ~17% разброса долей; остальные ~83% — **firm×cell-гетерогенность**
  (бренд/дистрибуция/лояльность в конкретной ячейке), которой параметрической
  модели не хватает (полный firm×cell = 144 параметра на 504 набл., §14.4).

**Фикс — заякорить прогноз на наблюдаемый спрос:** он несёт всю
firm×cell-гетерогенность «бесплатно» (достаточная статистика по последнему
кварталу вместо 144 дамми). Модель отвечает лишь за **причинный сдвиг** от смены
решений (рычаг). При неизменных решениях прогноз схлопывается в сезонный наив —
сильный бейзлайн, а не тянет всех к средней по ячейке (регрессия к среднему из
§14.3 устранена по построению).

## Ключевые наблюдения

1. **Стадия 1 (доля) сильна везде:** MAE 0.5–1.3 п.п., all-8 oracle 0.48–1.09.
   Логит воспроизводит наблюдаемые доли на всех 6 группах и обеих эрах (2017–2026).
2. **Стадия 2 (объём) хороша:** MAPE 3–6%.
3. **Сквозной спрос теперь бьёт наив в агрегате** (8.06% vs 8.77%) и на большинстве
   групп — против прежних 15.9%. Основной драйвер — заякоривание (уровень), рычаг
   даёт дополнительный выигрыш там, где решения менялись.
4. **Гетерогенность по группам сохраняется:** гр.11/31 сильно бьют наив
   (5.95/5.81% vs 10.49/8.01%); гр.13 (10.30 vs 8.73) и гр.2 (12.76 vs 11.04) всё
   ещё чуть проигрывают — 2019-эра / малая выборка (§15: пер-групповой рычаг OOS
   не обобщается). Демпфер k=0.5 — компромисс: смягчает провал gr.13/gr.2, почти
   не теряя на gr.11/31 (полный рычаг k=1.0 даёт агрегат 7.53%, но провал gr.13
   до 11.92% и gr.2 до 14.57%).

## Оговорки

- **In-sample:** модель доли обучается на тех же отчётах, что оценивает. Строгий
  OOS хуже (проверено ранее: train=гр.13 → test=гр.11 даёт realistic 27% vs 14%).
- **Малые выборки:** гр.31 и гр.49 — по 1 переходу (9 ячеек); гр.13 — 2. Цифры
  ориентировочные.
- **Демпфер рычага `--lever-k`:** дефолт 0.5 (полная эластичность логита
  переоценена и OOS не обобщается, §14.5/§15). k=1.0 — полный модельный рычаг
  (лучший агрегат, но менее робастен), k=0.0 — чистый сезонный наив. Рычаг
  ограничен клипом `[1/3, 3]` (защита от взрыва доли при вырожденном фите).
- **Усадка наклона:** λ=1000 (2026) / 1e6 (гр.2) → полный пулинг; global slope:
  −3.19 (2026), −1.93 (гр.2).
- **Realistic vs oracle:** близки → незнание ходов конкурентов вносит малую
  ошибку; основной сигнал — заякоренный уровень + собственный рычаг.

## Воспроизведение

```bash
# 2026 все группы (дефолт k=0.5)
uv run gmc-forecaster backtest \
  --reports data/W115*.xls data/W131*.xls data/W318*.xls data/W496*.xls \
  --history data/Hst*.xlsx
# то же с разбивкой по 9 ячейкам (+ sibling <out>.percell.csv при --out)
uv run gmc-forecaster backtest \
  --reports data/W115*.xls data/W131*.xls data/W318*.xls data/W496*.xls \
  --history data/Hst*.xlsx --per-cell
# сравнить силу рычага
uv run gmc-forecaster backtest --reports data/W115*.xls … --lever-k 1.0   # полный
uv run gmc-forecaster backtest --reports data/W115*.xls … --lever-k 0.0   # = сез.наив
# по-группно (пример гр.11)
uv run gmc-forecaster backtest --reports data/W115*.xls --history data/Hst*.xlsx
# гр.49: самообучение вырождено (Q2 без долей) — обучать на всех 2026-группах
uv run gmc-forecaster backtest --reports data/W496*.xls \
  --train data/W115*.xls data/W131*.xls data/W318*.xls data/W496*.xls \
  --history data/Hst*.xlsx
# гр.2 (2019) — без истории
uv run gmc-forecaster backtest --reports data/6.xls data/7.xls data/8.xls data/9.xls
```
