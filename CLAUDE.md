# gmc-forecaster

Прогноз спроса (ЗАКАЗЫ) в бизнес-симуляции Global Management Challenge.
Двухшаговая модель «доля рынка → спрос». Единица прогноза — **9 ячеек**:
3 продукта × 3 канала (EAEU/ЕАЭС, ASEAN/АСЕАН, INT/Интернет).

Почему share-модель, а не прямой own-demand, весь дата-анализ и нюансы данных
(спрос ≠ продажи ≠ доля; доля эндогенна и течёт как признак) — в
[docs/FINDINGS.md](docs/FINDINGS.md). **Важно:** FINDINGS.md — исследовательский
журнал, местами отстаёт от кода (см. «Расхождения»). При конфликте — код.

## Запуск

```bash
uv sync                 # окружение
make validate           # ruff format + ruff check + mypy --strict + pytest
uv run gmc-forecaster --help
```

Три подкоманды: `forecast` (прогноз под решениями игрока), `backtest` (оценка
на истории) и `cost` (полная себестоимость + contribution margin по 9 ячейкам).

```bash
# «крутить цены/рекламу»: правишь решения на листе 'Your decisions' в --current
uv run gmc-forecaster forecast \
  --current  data/W115264.xls \
  --train    data/W1152*.xls data/W1312*.xls \
  --history  data/Hst*.xlsx --out forecast.csv

# оценка качества модели по смежным парам кварталов
uv run gmc-forecaster backtest \
  --reports data/W1152*.xls data/W1312*.xls \
  --history data/Hst*.xlsx --out backtest.csv

# себестоимость РЕШЕНИЯ по 9 ячейкам (объём/цены/реклама/комиссии/смены/станки —
# с листа 'Your decisions'; --train/--history — кварталы своей фирмы для
# калибровки yield/спот-цены; --forecast — продажи для базы «прогноз»)
uv run gmc-forecaster cost \
  --current data/W115264.xls \
  --train   data/W11526[1-3].xls --out cost.csv
uv run gmc-forecaster cost --current data/W115264.xls \
  --train data/W11526[1-3].xls --forecast forecast.csv   # + база «прогноз»
```

Решения (цены по 9 ячейкам, реклама, **пер-канальные** число/комиссия
дистрибьюторов EAEU/ASEAN/INT) читаются **прямо с первого листа
`'Your decisions'`** файла `--current` (`parse_decisions`, позиционная схема
`DEC_SCHEMA`, языконезависимо) — json-сценарий убран. Правишь what-if решения
в этом листе Excel → `forecast`. Реклама → множитель бюджета компании-уровня;
дистрибьюторы действуют на все 3 продукта канала, наблюдаемы только у своей
фирмы → множитель собственной привлекательности `A_own` в логите доли. Сила
эффекта — коэф. `k_n`/`k_comm` (дефолт 0.15/0.10, оверрайд `--dist-k-n`/
`--dist-k-comm`). **Найм дистрибьюторов отложен на квартал**: охват лишь с
Q_next+1, поэтому на прогноз Q_next они **не влияют** (цена/реклама влияют
сразу); отложенный вклад — в колонке **Δ_дистриб_след_%**. Квартал прогноза =
следующий за кварталом в `--current`.

Прогноз **заякорен** на наблюдаемый спрос: `спрос_сцен = спрос_текущ ×
сезонность × рычаг`, рычаг = модельное отношение доли (сцен/база),
демпфированное `--lever-k` (дефолт 0.5; 1 = полный рычаг, 0 = чистый сезонный
наив), клип `[1/3,3]`. Наблюдаемый спрос несёт всю firm×cell-гетерогенность,
которую логит доли не восстанавливает → прежний абсолютный `доля × объём` давал
15-20% MAPE ([docs/BACKTEST.md](docs/BACKTEST.md)), заякоренный ~8% (бьёт наив).
Колонка **Δ_рычаг_%** = рычаг−1, изолирует эффект решения (неизменные решения =
0 → спрос_сцен = сез. наив). После таблицы `forecast` печатает **коэффициенты
модели доли** со значимостью (ст. ошибка, t, p, звёзды) + R²/n/edf/λ и строку
смысла β_price. Флаг `--coeffs {key,full,none}` (дефолт `key`: ключевые признаки
+ наклоны u_g; `full` — все FE ячейка/группа/фирма; `none` выключает). При
`--out` таблица коэффициентов дублируется в sibling `<out>.coef.csv`.

## Архитектура (`src/gmc_forecaster/`)

- **parser.py** — три парсера, все языко/версия-независимы (позиционные схемы),
  движок `engine="calamine"` (читает и BIFF `.xls`, и strict-OOXML `.xlsx`
  истории, где openpyxl не видит листов; он же в `load_panel`):
  - `parse_report(path)` → `(meta, df по 9 ячейкам)` с **листа `W`** (плоский
    числовой экспорт, схема `SCHEMA`, `off = 3*(product-1) + ch`). `_num`/
    `_stars` — нормализация ячеек/рейтинга.
  - `parse_decisions(path)` → dict решений с **листа `'Your decisions'`**
    (`DEC_SCHEMA`): `price{ячейка}`, `plan{ячейка}` (план поставок = `planned`),
    `assembly_min{продукт}` (факт. время сборки), `adspend_total` (тыс.ерз),
    `dist_n{канал}`, `dist_comm{канал}`, `shifts`, `hire`, `wage`, `mach_buy`/
    `mach_sell`; пустая ячейка → None (наследует текущее из `W`).
  - `parse_costs(path)` → `CostReport` с листов `'Resources and products'` и
    `'Financial statements'` (`RES_SCHEMA`/`FIN_SCHEMA`, продукты в столбцах
    20/22/24): выпуск/брак/полуфабрикаты, часы сборщиков/станков, эффективность,
    расход/закупка сырья, все статьи накладных и P&L. Требует компанию 1..8.
- **model.py** — ядро:
  - `load_panel(paths)` → панель по 8 компаниям (price/share/adspend/rating) +
    опция `share_out = 100 − Σ`.
  - `ShareModel` — **стадия 1**: логит доли через инверсию Берри `ln(sᵢ/s₀) =
    β·X + FE(ячейка)+FE(группа)+FE(фирма) + (β_price+u_g)·log_price`. Оценка —
    **штрафованный МНК** (штраф только на u_g), `FEATURES = [log_price,
    log_adspend, rating]`. **Firm-эффект** (дамми company×group, центр. внутри
    группы) ловит фирменную гетерогенность → снижает регрессию-к-среднему; это
    уровень, не наклон → эластичность почти не меняется; незнакомая фирма → на
    групповой эффект. **Random slope `u_g`** — эластичность гетерогенна по
    группам (−2.5…−4.9): усадка к global (ridge), λ из leave-one-quarter-out CV
    (`ridge_`) → partial pooling (на текущих данных CV выбирает пулинг).
    `predict_shares` — MNL-замещение, **исключает неактивные** (`price≤0`, иначе
    клип 0→1 даст фантомно дешёвого игрока). `counterfactual` — доли при смене
    цены. `coef_summary()` → коэф. со **значимостью**: ст. ошибки из сэндвич-
    ковариации штраф. МНК `σ²·Ainv·XᵀX·Ainv` (для непенализуемых = OLS
    `σ²·(XᵀX)⁻¹`), σ² на edf=tr(Ainv·XᵀX), p — норм. приближение (`erfc`); для
    `u_g` значимость приблизит.
  - `fit_seasonality(hst)` → сезонные множители {Q:factor} по группе 0.
    `cell_volume` = own_sold/(own_share/100). **Стадия 2**: объём с сезонной
    поправкой (для калибровки/oracle-отката).
  - `damp_lever(ratio, k)` — демпф. (к 1.0 на долю 1−k) + клип `LEVER_CLIP=
    [1/3,3]` причинный рычаг; `LEVER_K=0.5` дефолт (общий forecast/backtest).
    **Итог заякорен:** `спрос_t × сезонность × damp_lever(доля_сцен/доля_база)`.
- **forecast.py** — `forecast(current, train, history, k_n?, k_comm?, lever_k?)`:
  собирает пайплайн, читает `parse_decisions(current)`, применяет к своей
  компании, ЗАЯКОРИВАЕТ (база = спрос_текущ × сезонность; сцен = база × рычаг).
  Возвращает df (спрос текущий/база/сцен, доля, Δ_рычаг) + `df.attrs`:
  `meta` (вкл. `lever_k`), `coef_summary`, `fit` (R²/n/edf/λ).
  **Режим 1-й итерации half-final** (`_forecast_history`, авто при `--current`
  = history-рамп: компания вне 1..8, свои доли `'Not requested'`, 8 «компаний»
  листа `W` — идентичные клоны): стадия 1 недоступна → бейзлайн = персистенция
  спроса, рычаг = **заимствованная собственная эластичность** из модели на
  `--train`: `Δln(спрос) ≈ β_price·Δln(цена) + β_adspend·Δln(реклама+1) +
  ln(dist_mult)`, демпфер `lever_k`. `meta["mode"]=="history"`, `доля_сцен_%`=
  None, CLI печатает предупреждение. `--train` обязателен (даёт β).
- **backtest.py** — `backtest(reports, train, history, lever_k?)`: прогон по
  смежным парам Q_t→Q_{t+1} внутри серии (группа+компания), сравнение с фактом.
  Раскладывает ошибку на стадию 1 (доля, MAE) и стадию 2 (объём, MAPE),
  **заякоренный сквозной спрос** (`_anchor_demand`; MAPE) против бейзлайнов
  (персистенция, сезонный наив), в двух режимах: **realistic** (конкуренты
  заморожены на Q_t) и **oracle** (факт рынка Q_{t+1}). Нет наблюдаемого спроса
  → откат на абсолютный доля×объём. Решения берутся из отчёта Q_{t+1}. Возвращает
  `(сводка, детализация)`. `per_cell_summary(df)` → 9 ячеек: доля-MAE (real/
  oracle), объём-MAPE, спрос-MAPE + бейзлайны (без метрики «все 8 компаний»).
- **cost.py** — `cost(current, train?, history?, yield_halflife?, demand?)` → df
  по 9 ячейкам (ед/цена/выручка, компоненты затрат, `себест_полн_ед`, `CM_ед`,
  `прибыль_полн_ед`, `прибыль_ячейка`) + `df.attrs` (meta/пулы/yields/сверка).
  **Себестоимость РЕШЕНИЯ**: объём/цены/реклама/комиссии/смены/парк станков — с
  листа `'Your decisions'`, не из фактов прошлого квартала. **Оборотная база —
  план поставок** (`planned`, строки 23–25), не `sold` (убирает «продажи со
  склада»; сверка материала +6.7 %→+2.0 %). Строится **снизу вверх** из констант
  мануала (сырьё 1/2/3, станко 60/75/120 мин, машино-час 8, ОТК 1/ед, аморт.
  2.5 %) и рыночных величин (спот-цена, зарплаты). **Время сборки — с листа
  решений** (row30; ≠ константа {100,150,300} для этой игры, сверено с
  `assembler_worked_h`); станко-минуты {60,75,120} верны. Тождества: `сырьё =
  Σ(произв−п/ф)×{1,2,3}`, `машино-часы = Σ(произв−п/ф)×{60,75,120}/60/эфф`.
  **Разнесение двухступенчатое**: материал → по единицам плана; конверсия (труд,
  эксплуатация станков, аморт., ОТК) — фикс-пул по сборщико-минутам на фактич.
  времени сборки; канал-пул → к каналам; overhead (реклама + прочее) → по
  плановой выручке. Смены (`dec['shifts']`, fallback `shifts_needed`) и парк
  станков — из решений; **труд-пул фикс из отчёта** (v1: реконструкция по ставке
  шумна). Три yield-поправки EWMA (`calibrate_yields`). **Себест/ед и CM/ед
  базонезависимы**. При `demand` (продано по ячейкам) — доп. колонки базы
  **прогноза** (`продано_прог`/`выручка_прог`/`прибыль_прог`; продано = min(спрос,
  план)). **Двухрежимная сверка** (`_plan_edited`): решения = исполненным → смета
  сходится с `COGS+накл` (inventory-шум); отредактированы → форвард-оценка,
  `применима=False`. CM = `цена − материал − п/ф − комиссия`. `unit_costs(...)` →
  `{ячейка: {full, cm}}` для `forecast`. Требует компанию 1..8 (иначе `ValueError`;
  `unit_costs` → None).
- **cli.py** — entry point `gmc-forecaster` с `forecast`/`backtest`/`cost`.
  `forecast` после таблицы печатает блок коэффициентов (`--coeffs key|full|none`,
  `_print_coeffs`), при `--out` пишет sibling `<out>.coef.csv`; таблица дополнена
  `себест_полн_ед`/`CM_ед`/`прибыль_ячейка` (нет производ. данных → пропуск с
  предупреждением). `backtest --per-cell` → 9-строчная таблица (`per_cell_summary`/
  `_print_per_cell`), при `--out` дублирует в `<out>.percell.csv`. `cost` печатает
  смету + yield + строку режима сверки; `--forecast <csv>` добавляет колонки базы
  прогноза (`_read_forecast_demand`); при `--out` пишет CSV.

## Данные (`data/`)

- `W<gg><c><yy><q>.xls` — отчёт: группа/компания/год/квартал.
  `W1152*` = гр.11 комп.5 (Q1–Q4'26); `W131*` = гр.13 комп.1 (Q1–Q3'26);
  `W318*` = гр.31 комп.8 (Q2–Q3'26); `W496*` = гр.49 комп.6 (Q2–Q3'26).
- `Hst*.xlsx` — history-кварталы группы 0 (компании-клоны) → только сезонность.
- `1Quarter..5Quarter` = гр.1 комп.1 (2017–18); `6..10.xls` = гр.2 комп.4
  (2018–19) — версии RU/PL/ES, проверка версионной устойчивости парсера `W`.
  **Гр.1 не экспортирует доли** (позиции 331+ пусты) → непригодна для стадии 1;
  гр.2 годна (файл `10` — терминальный квартал, долей нет). У старых игр 2017–19
  нет своей сезонности (история группы 0 — 2024–26, неприменима).
- `gmc_w_schema.json` — выведенная схема листа `W` (справочно).

## Расхождения кода с docs/FINDINGS.md

- Имена `gmc_parser_flat.py`/`gmc_share_model.py`/`gmc_predict.py` устарели — это
  модули пакета выше + CLI `gmc-forecaster`.
- `scenario.json` **убран**: решения с листа `'Your decisions'` (`parse_decisions`),
  квартал — из `--current`.
- Именованного парсера нет: лист `W` (`parse_report`) и лист решений
  (`parse_decisions`) ведёт позиционная схема.
- Ридер — единый `engine="calamine"`, не автовыбор xlrd/openpyxl (§13): история
  в strict-OOXML, openpyxl её не читает.

- Python ≥3.14, `mypy --strict`, ruff line-length 79. Комментарии/вывод — русский.
