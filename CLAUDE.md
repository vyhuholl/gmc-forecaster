# gmc-forecaster

Прогноз спроса (ЗАКАЗЫ) в бизнес-симуляции Global Management Challenge. Двухшаговая модель «доля рынка → спрос». Единица прогноза — **9 ячеек**: 3 продукта × 3 канала (EAEU/ЕАЭС, ASEAN/АСЕАН, INT/Интернет). Обоснование share-модели, дата-анализ и нюансы (спрос ≠ продажи ≠ доля; доля эндогенна и течёт как признак) — в [docs/FINDINGS.md](docs/FINDINGS.md). **Важно:** FINDINGS.md — исследовательский журнал, местами отстаёт от кода (см. «Расхождения»); при конфликте — код.

## Запуск

Три подкоманды: `forecast` (прогноз под решениями), `backtest` (оценка на истории), `cost` (полная себестоимость + contribution margin по 9 ячейкам).
```bash
uv sync                 # окружение
make validate           # ruff format + ruff check + mypy --strict + pytest

# «крутить цены/рекламу»: правишь решения на листе 'Your decisions' в --current
uv run gmc-forecaster forecast --current data/W115264.xls \
  --train data/W1152*.xls data/W1312*.xls \
  --history data/Hst*.xlsx --out forecast.csv

# оценка качества модели по смежным парам кварталов
uv run gmc-forecaster backtest --reports data/W1152*.xls data/W1312*.xls \
  --history data/Hst*.xlsx --out backtest.csv

# себестоимость РЕШЕНИЯ по 9 ячейкам (--train/--history — кварталы своей фирмы
# для калибровки yield/спот-цены; --forecast <csv> — продажи для базы «прогноз»)
uv run gmc-forecaster cost --current data/W115264.xls \
  --train data/W11526[1-3].xls --out cost.csv
```

## Модель прогноза

**Решения** (цены по 9 ячейкам, реклама, **пер-канальные** число/комиссия дистрибьюторов) читаются **прямо с листа `'Your decisions'`** `--current` (`parse_decisions`, позиционная `DEC_SCHEMA`, языконезависимо). Правишь what-if в Excel → `forecast`; квартал прогноза = следующий за `--current`. Реклама → множитель бюджета; дистрибьюторы (все 3 продукта канала, наблюдаемы лишь у своей фирмы) → множитель `A_own` в логите, сила `k_n`/`k_comm` (дефолт 0.15/0.10, оверрайд `--dist-k-n`/`--dist-k-comm`). **Найм отложен на квартал** (охват с Q_next+1) → на прогноз Q_next не влияет (цена/реклама — сразу); отложенный вклад — колонка **Δ_дистриб_след_%**.

**Якорь на наблюдаемый спрос:** `спрос_сцен = спрос_текущ × сезонность × рычаг × авто-уровень`, рычаг = модельное отношение доли (сцен/база), демпф. `--lever-k` (дефолт 0.5; 1 = полный, 0 = сезонный наив), клип `[1/3,3]`. Наблюдаемый спрос несёт всю firm×cell-гетерогенность, которую логит не восстанавливает → прежний абсолютный `доля × объём` давал 15-20% MAPE ([docs/BACKTEST.md](docs/BACKTEST.md)), заякоренный ~8%. Колонка **Δ_рычаг_%** = рычаг−1, изолирует эффект решения.

**Сезонность `--seasonality {auto,market,history,none}`** (дефолт `auto`). `market` — рынок из `--train` (рынок ячейки = `свой_спрос/своя_доля`, убирает вариацию своей доли; `fit_market_seasonality`, тренд бунделится в полный множитель кв→кв). `history` — объём группы-0 `--history` (`fit_seasonality`, тренд отделён). `none` — 1. `auto` — market если train ≥2 кв., иначе history, иначе none.
**Важно:** в half-final группа-0 `--history` — РАМП входа, не сезонность (ложный Q3-пик) → `auto` предпочитает train-рынок. Бэктест: `market` бьёт всё — зрелые 9.6% (history 56.9%! none 10.2%), рамп gr.17 OOS 19.4%.

**Уровень доли `--mode {auto,anchored,absolute}`** (дефолт `auto`). Якорь верен, пока firm×cell-гетерогенность **персистентна** (зрелая фирма); на **рампе/входе** набл. доля « модельной `доля_база`, якорь замораживает транзиент → занижение. Бленд: `factor = gap^(w−1)`, `gap = набл_доля/доля_база` (клип `[0.1,10]`), `w∈[0,1]`: `w=1`→anchored, `w=0`→absolute (`доля_модель × рынок`). **`w` — по ДИНАМИКЕ разрыва в своей серии, не по уровню** (`gap_persistence`): AR(1) без константы `ln gap_{t+1}=w·ln gap_t`, ridge-усадка к уровневому приору `gap_weight=exp(−(ln gap)²/2τ²)` силой λ=τ² (`--gap-tau` 0.5). **Устойчивый разрыв** (firm-FE один на фирму, не firm×cell → модель не знает слабую ячейку) → `w→1` якорь; **рамп** (gap→1) → `w→0` модель; истории <2 кв. → чистый приор. Уровневый `w=gap_weight` путал эти случаи → ×3 (гр.15 к.2 АСЕАН3: разрыв стоит ≈0.3, спрос-MAPE 148%, агрегат 32.8%; с историей 13.4%≈`anchored` 12.7%). Бэктест: зрелые `auto`≈`anchored`, рамп gr.17 OOS `auto` 20.0% (`anchored` 24.0%). Колонки **разрыв_доли**/**вес_якоря**; свод/CLI печатают глубину истории (0→приор), CLI — ячейки с малым весом якоря. История: `forecast` из `--train`+`--current`, `backtest` из `--reports`/`--train` со срезом строго до Q_now.

## Архитектура (`src/gmc_forecaster/`)

Все парсеры языко/версия-независимы (позиционные схемы), движок `engine="calamine"` (читает и BIFF `.xls`, и strict-OOXML `.xlsx` истории, где openpyxl не видит листов; он же в `load_panel`).
- **parser.py** — три парсера:
  - `parse_report(path)` → `(meta, df по 9 ячейкам)` с **листа `W`** (плоский числовой экспорт, схема `SCHEMA`, `off = 3*(product-1) + ch`). `_num`/`_stars` — нормализация ячеек/рейтинга.
  - `parse_decisions(path)` → dict с **листа `'Your decisions'`** (`DEC_SCHEMA`): `price{ячейка}`, `plan{ячейка}` (= `planned`), `assembly_min{продукт}` (факт. время сборки), `adspend_total`, `dist_n/dist_comm{канал}`, `shifts`, `hire`, `wage`, `mach_buy/mach_sell`; пустая ячейка → None (наследует из `W`).
  - `parse_costs(path)` → `CostReport` с листов `'Resources and products'` и `'Financial statements'` (`RES_SCHEMA`/`FIN_SCHEMA`, продукты в столбцах 20/22/24): выпуск/брак/п-ф, часы сборщиков/станков, эффективность, расход/ закупка сырья, накладные и P&L. Требует компанию 1..8.
- **model.py** — ядро:
  - `load_panel(paths)` → панель по 8 компаниям (price/share/adspend/rating), опция `share_out = 100 − Σ`. Числовые колонки — через `to_numeric` (пусто → NaN; иначе целиком пустая колонка из одного файла даёт dtype=object, роняет `np.log`).
  - `ShareModel` — **стадия 1**: логит доли (инверсия Берри) `ln(sᵢ/s₀) = β·X + FE(ячейка+группа+фирма) + (β_price+u_g)·log_price`, оценка — **штрафованный МНК** (штраф только на u_g), `FEATURES = [log_price, log_adspend, rating]`.
    **Firm-эффект** (дамми company×group, центр. внутри группы) ловит фирменную гетерогенность уровня (не наклон; незнакомая фирма → на групповой эффект).
    **Random slope u_g** — эластичность гетерогенна по группам (−2.5…−4.9): усадка к global (ridge), λ из leave-one-quarter-out CV (`ridge_`). **Неизвестная реклама ≠ 0**: строки `adspend=NaN` вне фита (`n_no_adspend`; нули учили бы β_adspend «рекламы не было»), на предсказании — среднее `_ladspend_mean`, иначе `log(0+1)=0` обвалит ячейку во внешнюю опцию (доли не ренормируются: 0.02% против факта 12%); так же рейтинг (`_rating_mean`). `predict_shares` — MNL-замещение, **исключает неактивные** (`price≤0`). `counterfactual` — доли при смене цены. `coef_summary()` → значимость (сэндвич-ковариация штраф. МНК, p через `erfc`; u_g приблиз.).
  - `fit_seasonality(hst)` → {Q:factor} по группе 0 (тренд отделён); `fit_market_seasonality(reports)` → по РЫНКУ (=свой_спрос/своя_доля; тренд бунделится); `resolve_seasonality(mode,market,hist)` → `(факторы|None,источник)`, `auto` предпочитает train-рынок. `cell_volume` = own_sold/(own_share/100).
  - `damp_lever(ratio, k)` — демпф. рычаг (к 1.0 на долю 1−k) + клип `LEVER_CLIP=[1/3,3]`; `LEVER_K=0.5`. Авто-уровень (см. `--mode`): `gap_weight`(уровневый приор), `gap_persistence`(AR(1)+ridge по истории), `anchor_weight`(w по mode), `level_factor(набл,база,mode,tau,gap_hist)` → `gap^(w−1)`; `own_share`(доля своей фирмы), `gap_panel`/`gaps_before`/`gap_history`(история разрыва серии). `MODES=(auto,anchored,absolute)`.
- **forecast.py** — `forecast(current, train, history, k_n?, k_comm?, lever_k?, mode="auto", gap_tau?, seasonality="auto")`: `resolve_seasonality` выбирает источник; читает `parse_decisions`, применяет к своей компании, ЗАЯКОРИВАЕТ (база = спрос_текущ × сезонность × `level_factor`; сцен = база × рычаг). df (спрос текущ/база/сцен, доля, Δ_рычаг, **разрыв_доли**, **вес_якоря**) + `attrs` (`meta` вкл. lever_k/mode/gap_tau/gap_hist, `coef_summary`, `fit`). **Режим 1-й итерации half-final** (`_forecast_history`, авто когда `--current` = history-рамп: компания вне 1..8, доли `'Not requested'`, 8 клонов листа `W`): стадия 1 недоступна → бейзлайн = персистенция спроса, рычаг = **заимствованная эластичность** из `--train` (`Δln(спрос) ≈ β_price·Δln(цена) + β_adspend·Δln(рек +1)`). `meta["mode"]=="history"`, `доля_сцен_%`=None, `--train` обязателен.
- **backtest.py** — `backtest(reports, train, history, lever_k?, mode="auto", gap_tau?, seasonality="auto")`: прогон по смежным парам Q_t→Q_{t+1} внутри серии (группа+компания), сравнение с фактом. Раскладывает ошибку на стадию 1 (доля, MAE) и стадию 2 (объём, MAPE), **заякоренный спрос** (`_anchor_demand`, тот же `level_factor(mode)` + история разрыва серии со срезом до Q_now; MAPE) против бейзлайнов (персистенция, сезонный наив), в режимах **realistic** (конкуренты на Q_t) и **oracle** (факт рынка Q_{t+1}). Нет набл. спроса → откат на абсолютный доля×объём. Решения — из отчёта Q_{t+1}. Если Q_{t+1} без рекламы/рейтинга
  (финальный отчёт), они переносятся с Q_t (`_carry_forward`) → oracle там «по ценам», счётчики в сводке (`ячеек_реклама_с_Q_now`, `строк_вне_фита_без_рекламы`, `история_разрыва_кв`); realistic не затронут. `per_cell_summary(df)` → 9 ячеек (доля-MAE real/oracle, объём/спрос-MAPE + бейзлайны).
- **cost.py** — `cost(current, train?, history?, yield_halflife?, demand?)` → df по 9 ячейкам (ед/цена/выручка, компоненты затрат, `себест_полн_ед`, `CM_ед`, `прибыль_полн_ед`, `прибыль_ячейка`) + `attrs`. **Себестоимость РЕШЕНИЯ** (объём/цены/реклама/комиссии/смены/парк — с листа `'Your decisions'`), снизу вверх из констант мануала (сырьё 1/2/3, станко 60/75/120 мин, машино-час 8, ОТК 1/ед, аморт. 2.5%) и рынка (спот-цена, зарплаты). **Оборотная база — план поставок** (`planned`), не `sold` (сверка материала +6.7%→+2.0%). **Время сборки — с листа решений** (row30; ≠ константа {100,150,300}, сверено с `assembler_worked_h`); станко-минуты {60,75,120} верны. **Разнесение двухступенчатое**: материал → по единицам плана; конверсия (труд/станки/аморт/ОТК) — фикс-пул по сборщико-минутам; канал-пул → к каналам; overhead → по плановой выручке. Смены (`dec['shifts']`, fallback `shifts_needed`)/парк — из решений. Три yield-поправки EWMA (`calibrate_yields`). **Себест/ед и CM/ед базонезависимы**. При `demand` — колонки базы прогноза (`продано_прог` = min(спрос, план)). **Двухрежимная сверка** (`_plan_edited`): решения = исполненным → сходится с `COGS+накл`; отредактированы → форвард-оценка, `применима=False`. CM = `цена − материал − п/ф − комиссия`; `unit_costs(...)` → `{ячейка:{full,cm}}` для `forecast`. Требует компанию 1..8.
- **cli.py** — entry point `forecast`/`backtest`/`cost`; `forecast`/`backtest` принимают `--mode`/`--gap-tau`/`--seasonality` (дефолты auto). `forecast`: коэффициенты (`--coeffs`), таблица + `разрыв_доли`/`вес_якоря`/себест/CM/прибыль; при `--out` — sibling `<out>.coef.csv`. `backtest --per-cell` → 9 строк (`<out>.percell.csv`). `cost`: смета + yield + режим сверки, `--forecast <csv>` → колонки базы прогноза; при `--out` пишет CSV.

## Данные (`data/`)

- `W<gg><c><yy><q>.xls` — отчёт: группа/компания/год/квартал. `W1152*` = гр.11 комп.5 (Q1–Q4'26); `W131*` = гр.13 комп.1 (Q1–Q3'26); `W318*` = гр.31 комп.8 (Q2–Q3'26); `W496*` = гр.49 комп.6 (Q2–Q3'26); `W172*` = гр.17 комп.2 (Q3–Q4'26 + Q1–Q2'27, своя фирма). **`W172272` — финальный отчёт: цены и доли есть, рекламы и рейтинга нет** (пусты у всех 8) → в `--reports` бэктеста годен, в фит строки не попадут.
- `Hst*.xlsx` — history-кварталы группы 0. **В half-final (`data/half-final/`) — РАМП входа по каналам, НЕ сезонные клоны** → `fit_seasonality` даёт ложный Q3-пик; бери `--seasonality market`. Старая группа-0 (сезонные клоны) — валидный источник `history`.
- `1Quarter..5Quarter` = гр.1 (2017–18); `6..10.xls` = гр.2 (2018–19) — версии RU/PL/ES, проверка устойчивости парсера `W`. **Гр.1 не экспортирует доли** → непригодна для стадии 1; гр.2 годна. `gmc_w_schema.json` — схема `W` (справочно).
