# gmc-forecaster

Прогноз спроса (ЗАКАЗЫ) в бизнес-симуляции Global Management Challenge.
Двухшаговая модель «доля рынка → спрос». Единица прогноза — **9 ячеек**:
3 продукта × 3 канала (EAEU/ЕАЭС, ASEAN/АСЕАН, INT/Интернет). Почему
share-модель, а не прямой own-demand, дата-анализ и нюансы (спрос ≠ продажи ≠
доля; доля эндогенна и течёт как признак) — в [docs/FINDINGS.md](docs/FINDINGS.md).
**Важно:** FINDINGS.md — исследовательский журнал, местами отстаёт от кода (см.
«Расхождения»). При конфликте — код.

## Запуск

Три подкоманды: `forecast` (прогноз под решениями), `backtest` (оценка на
истории), `cost` (полная себестоимость + contribution margin по 9 ячейкам).
```bash
uv sync                 # окружение
make validate           # ruff format + ruff check + mypy --strict + pytest
uv run gmc-forecaster --help

# «крутить цены/рекламу»: правишь решения на листе 'Your decisions' в --current
uv run gmc-forecaster forecast --current data/W115264.xls \
  --train data/W1152*.xls data/W1312*.xls \
  --history data/Hst*.xlsx --out forecast.csv

# оценка качества модели по смежным парам кварталов
uv run gmc-forecaster backtest --reports data/W1152*.xls data/W1312*.xls \
  --history data/Hst*.xlsx --out backtest.csv

# себестоимость РЕШЕНИЯ по 9 ячейкам (объём/цены/реклама/комиссии/смены/станки —
# с листа 'Your decisions'; --train/--history — кварталы своей фирмы для
# калибровки yield/спот-цены; --forecast <csv> — продажи для базы «прогноз»)
uv run gmc-forecaster cost --current data/W115264.xls \
  --train data/W11526[1-3].xls --out cost.csv
```

## Модель прогноза

**Решения** (цены по 9 ячейкам, реклама, **пер-канальные** число/комиссия
дистрибьюторов) читаются **прямо с листа `'Your decisions'`** `--current`
(`parse_decisions`, позиционная схема `DEC_SCHEMA`, языконезависимо; json-сценарий
убран). Правишь what-if в Excel → `forecast`. Квартал прогноза = следующий за
`--current`. Реклама → множитель бюджета; дистрибьюторы (все 3 продукта канала,
наблюдаемы лишь у своей фирмы) → множитель `A_own` в логите, сила — `k_n`/`k_comm`
(дефолт 0.15/0.10, оверрайд `--dist-k-n`/`--dist-k-comm`). **Найм дистрибьюторов
отложен на квартал** (охват с Q_next+1) → на прогноз Q_next не влияет (цена/реклама
— сразу); отложенный вклад — колонка **Δ_дистриб_след_%**.

**Якорь на наблюдаемый спрос:** `спрос_сцен = спрос_текущ × сезонность × рычаг ×
авто-уровень`, рычаг = модельное отношение доли (сцен/база), демпф. `--lever-k`
(дефолт 0.5; 1 = полный рычаг, 0 = чистый сезонный наив), клип `[1/3,3]`.
Наблюдаемый спрос несёт всю firm×cell-гетерогенность, которую логит доли не
восстанавливает → прежний абсолютный `доля × объём` давал 15-20% MAPE
([docs/BACKTEST.md](docs/BACKTEST.md)), заякоренный ~8%. Колонка **Δ_рычаг_%** =
рычаг−1, изолирует эффект решения (неизменные решения = 0 → сез. наив).

**Сезонность `--seasonality {auto,market,history,none}`** (дефолт `auto`).
`market` — рынок из отчётов `--train` (рынок ячейки = `свой_спрос/своя_доля`,
убирает вариацию своей доли; `fit_market_seasonality`, тренд бунделится — полный
множитель кв→кв, для одногодичного 2026 train). `history` — объём группы-0
`--history` (`fit_seasonality`, тренд отделён). `none` — 1. `auto` — market если
train ≥2 кв., иначе history, иначе none. **Важно:** в half-final группа-0
`--history` — РАМП входа по каналам, не сезонность (ложный Q3-пик) → `auto`
предпочитает train-рынок. Бэктест: `market` бьёт всё — зрелые 9.6% (history 56.9%!
none 10.2%), рамп gr.17 OOS 19.4%.

**Уровень доли `--mode {auto,anchored,absolute}`** (дефолт `auto`). Якорь верен,
пока firm×cell-гетерогенность **персистентна** (зрелая фирма). На **рампе/входе**
набл. доля « модельной `доля_база`, якорь замораживает транзиент → занижение
(half-final gr.17: АСЕАН в 4-8×). Признак — **разрыв `gap = набл_доля/доля_база`**:
gap≈1 равновесие (якорь), gap далёк — вне режима (модель). Бленд: `factor =
gap^(w−1)`, `w = exp(−(ln gap)²/2τ²)` (`--gap-tau` дефолт 0.5, gap клип `[0.1,10]`):
`w=1`→`factor=1` (anchored), `w=0`→`factor=доля_база/набл` (absolute = `доля_модель
× рынок`); `auto` пер-ячейке. Бэктест: зрелые `auto`≈`anchored` (9.9→10.2%), рамп
gr.17 вдвое лучше (48→24%, absolute 21%). Колонки **разрыв_доли**(gap)/
**вес_якоря**(w); на рампе CLI печатает ячейки вне равновесия, `anchored` = вес=1.

## Архитектура (`src/gmc_forecaster/`)

Все парсеры языко/версия-независимы (позиционные схемы), движок
`engine="calamine"` (читает и BIFF `.xls`, и strict-OOXML `.xlsx` истории, где
openpyxl не видит листов; он же в `load_panel`).
- **parser.py** — три парсера:
  - `parse_report(path)` → `(meta, df по 9 ячейкам)` с **листа `W`** (плоский
    числовой экспорт, схема `SCHEMA`, `off = 3*(product-1) + ch`). `_num`/`_stars`
    — нормализация ячеек/рейтинга.
  - `parse_decisions(path)` → dict с **листа `'Your decisions'`** (`DEC_SCHEMA`):
    `price{ячейка}`, `plan{ячейка}` (= `planned`), `assembly_min{продукт}` (факт.
    время сборки), `adspend_total`, `dist_n/dist_comm{канал}`, `shifts`, `hire`,
    `wage`, `mach_buy/mach_sell`; пустая ячейка → None (наследует из `W`).
  - `parse_costs(path)` → `CostReport` с листов `'Resources and products'` и
    `'Financial statements'` (`RES_SCHEMA`/`FIN_SCHEMA`, продукты в столбцах
    20/22/24): выпуск/брак/п-ф, часы сборщиков/станков, эффективность,
    расход/закупка сырья, статьи накладных и P&L. Требует компанию 1..8.
- **model.py** — ядро:
  - `load_panel(paths)` → панель по 8 компаниям (price/share/adspend/rating),
    опция `share_out = 100 − Σ`. Числовые колонки — через `to_numeric` (пусто →
    NaN): панель из ОДНОГО файла с целиком пустой колонкой иначе получала
    dtype=object и роняла `np.log`.
  - `ShareModel` — **стадия 1**: логит доли (инверсия Берри) `ln(sᵢ/s₀) = β·X +
    FE(ячейка+группа+фирма) + (β_price+u_g)·log_price`, оценка — **штрафованный
    МНК** (штраф только на u_g), `FEATURES = [log_price, log_adspend, rating]`.
    **Firm-эффект** (дамми company×group, центр. внутри группы) ловит фирменную
    гетерогенность → снижает регрессию-к-среднему (уровень, не наклон; незнакомая
    фирма → на групповой эффект). **Random slope u_g** — эластичность гетерогенна
    по группам (−2.5…−4.9): усадка к global (ridge), λ из leave-one-quarter-out CV
    (`ridge_`). **Неизвестная реклама ≠ 0**: строки `adspend=NaN` вне фита
    (`n_no_adspend`; нули учили бы β_adspend выдуманному «рекламы не было»), на
    предсказании — среднее обучения `_ladspend_mean`, иначе `log(0+1)=0` у всех
    компаний ячейки обвалит её во внешнюю опцию (доли не ренормируются к 100:
    замер 0.02% против факта 12%). Так же рейтинг (`_rating_mean`).
    `predict_shares` — MNL-замещение, **исключает неактивные** (`price≤0`).
    `counterfactual` — доли при смене цены. `coef_summary()` → значимость: ст.
    ошибки из сэндвич-ковариации штраф. МНК `σ²·Ainv·XᵀX·Ainv` (непенализуемые =
    OLS), σ² на edf=tr(Ainv·XᵀX), p — `erfc`; u_g приблиз.
  - `fit_seasonality(hst)` → {Q:factor} по группе 0 (объём, тренд отделён).
    `fit_market_seasonality(reports)` → {Q:factor} по РЫНКУ (=свой_спрос/своя_доля;
    тренд бунделится). `resolve_seasonality(mode, market_paths, hist_paths)` →
    `(факторы|None, источник)`; `auto` предпочитает train-рынок. `cell_volume` =
    own_sold/(own_share/100).
  - `damp_lever(ratio, k)` — демпф. (к 1.0 на долю 1−k) + клип
    `LEVER_CLIP=[1/3,3]`; `LEVER_K=0.5` дефолт. `gap_weight(gap, tau=0.5)` →
    `w=exp(−(ln gap)²/2τ²)`; `level_factor(набл, база, mode, tau)` → множитель
    уровня: `anchored`→1, `absolute`→`gap^−1`, `auto`→`gap^(w−1)` (gap клип
    `[0.1,10]`; нет/≤0 долей → 1.0). `MODES=(auto,anchored,absolute)`.
- **forecast.py** — `forecast(current, train, history, k_n?, k_comm?, lever_k?,
  mode="auto", gap_tau?, seasonality="auto")`: `resolve_seasonality` выбирает
  источник; читает `parse_decisions`, применяет к своей компании, ЗАЯКОРИВАЕТ
  (база = спрос_текущ × сезонность × `level_factor`; сцен = база × рычаг). df
  (спрос текущ/база/сцен, доля, Δ_рычаг, **разрыв_доли**, **вес_якоря**) + `attrs`
  (`meta` вкл. lever_k/mode/gap_tau, `coef_summary`, `fit`). **Режим 1-й итерации
  half-final** (`_forecast_history`, авто когда `--current` = history-рамп:
  компания вне 1..8, доли `'Not requested'`, 8 клонов листа `W`): стадия 1
  недоступна → бейзлайн = персистенция спроса, рычаг = **заимствованная
  эластичность** из `--train` (`Δln(спрос) ≈ β_price·Δln(цена) + β_adspend·Δln(рек
  +1) + ln(dist_mult)`, демпф. `lever_k`). `meta["mode"]=="history"`,
  `доля_сцен_%`=None, `--train` обязателен.
- **backtest.py** — `backtest(reports, train, history, lever_k?, mode="auto",
  gap_tau?, seasonality="auto")`: прогон по смежным парам Q_t→Q_{t+1} внутри серии
  (группа+компания), сравнение с фактом. Раскладывает ошибку на стадию 1 (доля,
  MAE) и стадию 2 (объём, MAPE), **заякоренный спрос** (`_anchor_demand`, тот же
  `level_factor(mode)`; MAPE) против бейзлайнов (персистенция, сезонный наив), в
  режимах **realistic** (конкуренты на Q_t) и **oracle** (факт рынка Q_{t+1}). Нет
  набл. спроса → откат на абсолютный доля×объём. Решения — из отчёта Q_{t+1}. Если
  Q_{t+1} без рекламы/рейтинга (финальный отчёт), они переносятся с Q_t
  (`_carry_forward`) → oracle там «по ценам», счётчики в сводке
  (`ячеек_реклама_с_Q_now`, `строк_вне_фита_без_рекламы`); realistic не затронут —
  он и так на панели Q_t. `per_cell_summary(df)` → 9 ячеек (доля-MAE real/oracle,
  объём/спрос-MAPE + бейзлайны).
- **cost.py** — `cost(current, train?, history?, yield_halflife?, demand?)` → df
  по 9 ячейкам (ед/цена/выручка, компоненты затрат, `себест_полн_ед`, `CM_ед`,
  `прибыль_полн_ед`, `прибыль_ячейка`) + `attrs`. **Себестоимость РЕШЕНИЯ**:
  объём/цены/реклама/комиссии/смены/парк — с листа `'Your decisions'`. **Оборотная
  база — план поставок** (`planned`), не `sold` (убирает продажи со склада; сверка
  материала +6.7%→+2.0%). Снизу вверх из констант мануала (сырьё 1/2/3, станко
  60/75/120 мин, машино-час 8, ОТК 1/ед, аморт. 2.5%) и рынка (спот-цена,
  зарплаты). **Время сборки — с листа решений** (row30; ≠ константа {100,150,300},
  сверено с `assembler_worked_h`); станко-минуты {60,75,120} верны. **Разнесение
  двухступенчатое**: материал → по единицам плана; конверсия (труд/станки/аморт/ОТК)
  — фикс-пул по сборщико-минутам; канал-пул → к каналам; overhead (реклама+прочее)
  → по плановой выручке. Смены (`dec['shifts']`, fallback `shifts_needed`)/парк — из
  решений; труд-пул фикс из отчёта. Три yield-поправки EWMA (`calibrate_yields`).
  **Себест/ед и CM/ед базонезависимы**. При `demand` — колонки базы прогноза
  (`продано_прог` = min(спрос, план) и др.). **Двухрежимная сверка**
  (`_plan_edited`): решения = исполненным → сходится с `COGS+накл`; отредактированы
  → форвард-оценка, `применима=False`. CM = `цена − материал − п/ф − комиссия`;
  `unit_costs(...)` → `{ячейка:{full,cm}}` для `forecast`. Требует компанию 1..8.
- **cli.py** — entry point с `forecast`/`backtest`/`cost`. `forecast`/`backtest`
  принимают `--mode`/`--gap-tau`/`--seasonality` (дефолты auto). `forecast`: блок
  коэффициентов (`--coeffs`), таблица + `разрыв_доли`/`вес_якоря`/`себест_полн_ед`/
  `CM_ед`/`прибыль_ячейка`; при `--out` — sibling `<out>.coef.csv`. `backtest
  --per-cell` → 9 строк (`<out>.percell.csv`). `cost`: смета + yield + режим сверки,
  `--forecast <csv>` → колонки базы прогноза; при `--out` пишет CSV.

## Данные (`data/`)

- `W<gg><c><yy><q>.xls` — отчёт: группа/компания/год/квартал. `W1152*` = гр.11
  комп.5 (Q1–Q4'26); `W131*` = гр.13 комп.1 (Q1–Q3'26); `W318*` = гр.31 комп.8
  (Q2–Q3'26); `W496*` = гр.49 комп.6 (Q2–Q3'26); `W172*` = гр.17 комп.2 (Q3–Q4'26
  + Q1–Q2'27, своя фирма). **`W172272` — финальный отчёт: цены и доли есть,
  рекламы и рейтинга нет** (пусты у всех 8) → в `--reports` бэктеста годен, в фит
  строки не попадут.
- `Hst*.xlsx` — history-кварталы группы 0. **В half-final (`data/half-final/`) —
  РАМП входа по каналам, НЕ сезонные клоны** → `fit_seasonality` даёт ложный
  Q3-пик; бери `--seasonality market`. Старая группа-0 (сезонные клоны) — валидный
  источник `history`.
- `1Quarter..5Quarter` = гр.1 комп.1 (2017–18); `6..10.xls` = гр.2 комп.4
  (2018–19) — версии RU/PL/ES, проверка устойчивости парсера `W`. **Гр.1 не
  экспортирует доли** (позиции 331+ пусты) → непригодна для стадии 1; гр.2 годна;
  у игр 2017–19 нет своей сезонности. `gmc_w_schema.json` — схема `W` (справочно).
